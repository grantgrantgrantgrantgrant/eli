Setup Guide

ELI Home Setup

Personal GitHub account + iOS Shortcut — so you can fire off guide requests straight from Instagram or a random thought on the couch.

TL;DR

Create a free personal GitHub account → make a private repo → generate a PAT → build an iOS Shortcut that creates GitHub Issues from the share sheet or home screen. Requests route to either your work (public) or private pipeline depending on a one-tap choice.

Overview

How the System Works

One iOS Shortcut, two pipelines. When you share a reel or type a topic, the Shortcut asks "Work or Private?" and routes the request to the right GitHub repo as an Issue.

Your Phone
iOS Shortcut
Share sheet or home screen tap
↓ "Work or Private?"
Work Pipeline
eli
Public repo → GitHub Pages
Processed on Work Claude
Private Pipeline
eli-private
Private repo → local only
Processed on Home Claude
⚠️ The separation is absolute. Sensitive topics never touch your work GitHub account, work Claude, or any work system. Two accounts, two repos, two processing paths.
02 / 07

Part 1

Create Personal GitHub Account

This is your private account, completely separate from your work GitHub. It's free and takes 2 minutes.

1

Go to github.com/signup

Use your personal email (not work). Pick a username that doesn't link back to your work identity if privacy matters to you.

2

Create the private repo

Once you're in, click the + button (top right) → New repository

Repository name: eli-private
Visibility: Private ← this is critical
README: leave off
Everything else: leave as default

Click Create repository.

3

Generate a PAT for the private repo

Go to github.com/settings/personal-access-tokens/new (or Settings → Developer Settings → Personal Access Tokens → Fine-grained tokens → Generate new token)

Token name: eli-private-shortcut
Expiration: 90 days
Repository access: Only select repositories → eli-private
Permissions: Click "+ Add permissions" → search Issues → set to Read and write

Click Generate token. Copy the token and save it somewhere safe — you'll need it for the Shortcut. It starts with github_pat_...

💡 This PAT can only create issues on eli-private. It can't access any other repo, read your code, or do anything else. If it leaks, the worst someone could do is create junk issues on a private repo only you can see.

💡 Two PATs, two accounts, completely separate:
• eli-shortcut → work GitHub (generated at work) → Issues on eli (public)
• eli-private-shortcut → personal GitHub (generated at home) → Issues on eli-private (private)

Your work GitHub is never logged into from home. Your personal GitHub is never logged into at work.
04 / 07

Part 3

Build the iOS Shortcut

Open the Shortcuts app on your iPhone. Tap + to create a new Shortcut. Name it "ELI".

Add the following actions in order. Use the search bar at the bottom of the Shortcuts editor to find each action by name.

Action 1
Receive any input from Share Sheet
Tap "Any" and ensure URLs and Text are selected. This lets the Shortcut appear when you share a reel or webpage. It's the very first action and may appear automatically — if you see "Receive input from" at the top, you're good.
Action 2
Set variable url to Shortcut Input
Search for "Set Variable". Name the variable url. Set its value to Shortcut Input. This captures the shared link (if any).
Action 3
Choose from Menu: "Work" / "Private"
Search for "Choose from Menu". Set the prompt to "Route to:". Add two options: Work and Private.
Action 4 — inside "Work" branch
Ask for Input: "What do you want to know?"
Search for "Ask for Input". Set the prompt to "What do you want to know?". Set type to Text.
Action 5 — inside "Work" branch
Set variable note to Provided Input
Search for "Set Variable". Name it note, value is Provided Input (the answer from Action 4).
Action 6 — inside "Work" branch
Get Contents of URL (GitHub API — create Issue on eli)
Search for "Get Contents of URL".

URL: https://api.github.com/repos/grantgrantgrantgrantgrant/eli/issues
Method: POST
Headers:
  • Authorization → Bearer YOUR_WORK_PAT_HERE
  • Accept → application/vnd.github+json
Request Body: JSON
  • title → the note variable
  • body → the url variable
  • labels → (leave blank for now, or add ["request"])

Replace YOUR_WORK_PAT_HERE with the actual eli-shortcut PAT you generated.
Action 7 — inside "Work" branch
Show Notification: "✅ Work request queued"
Search for "Show Notification". Set title to ELI, body to Work request queued.
Actions 8–11 — inside "Private" branch
Repeat Actions 4–7 but for private repo
Same pattern: Ask for Input → Set variable → Get Contents of URL → Show Notification.

The only differences:
  • URL: https://api.github.com/repos/YOUR_PERSONAL_USERNAME/eli-private/issues
  • Authorization: Bearer YOUR_PRIVATE_PAT_HERE
  • Notification: "🔒 Private request queued"

Replace YOUR_PERSONAL_USERNAME and YOUR_PRIVATE_PAT_HERE with your personal GitHub details.
✅ That's the whole Shortcut. 11 actions total: receive input, capture URL, choose route, then each branch asks what you want to know and fires it to the right GitHub repo.

Part 4

Make It Easy to Trigger

The Shortcut now works from the share sheet (for reels/links). But you also want to launch it from the home screen for random thoughts. Here's how to get both.

1

Share sheet (already done)

Because Action 1 accepts input from the Share Sheet, the Shortcut automatically appears when you share a reel. If you don't see it, go to the share sheet → scroll right → Edit Actions → toggle ELI on.

2

Home screen icon

In the Shortcuts app, long-press the ELI Shortcut → Details → Add to Home Screen. Pick an icon and colour. Now you can tap it from your home screen — when there's no shared URL, the url variable will just be empty, and the Issue body will be blank. The title (your typed topic) is what matters.

3

Lock screen widget (optional)

On iOS 16+, you can add a Shortcuts widget to your lock screen. Hold the lock screen → Customize → add the Shortcuts widget → select ELI. One tap from locked phone.

D

Open Cowork → tell Claude "check my ELI queue" → Claude reads the GitHub Issues on the eli repo, builds the guides, pushes them to GitHub Pages, and closes the Issues.

E

Private topics at home

Open Claude Pro at home → tell Claude "check my private ELI queue" → process from the eli-private repo. Output stays on your home machine.
