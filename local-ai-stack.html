Internal Reference

Local AI Stack

Mac Mini + Ollama + OpenClaw — how it works, what it replaces, and what it doesn't.

TL;DR

A Mac Mini runs free, open-source AI models via Ollama. OpenClaw connects it to Teams as "RadMax." It does NOT use Claude/ChatGPT. It does NOT learn from conversations. Your data never leaves the building.

Why? Unlimited free AI for everyday tasks so the team saves paid tokens for complex work where quality matters.

Architecture

The Three Pieces

The setup has three layers. Each does exactly one job.

🖥️

Mac Mini (M4)

The hardware. A compact Apple computer on the office network. Its chip has unified memoryApple Silicon's architecture where CPU and GPU share the same RAM pool — critical for running large AI models. (GPU + RAM share one pool), ideal for AI. Acts as a dedicated always-on server.

🦙

Ollama

The AI engine. Free software on the Mac Mini. Downloads, manages, and runs open-source modelsAI models whose code/weights are publicly available for free — e.g. Llama (Meta), Mistral, DeepSeek. (Llama, Mistral, DeepSeek). Exposes an APIApplication Programming Interface — a "door" that lets one piece of software talk to another. that other software talks to.

🤖

OpenClaw

The chat router. Open-source software connecting Ollama to Teams, Slack, WhatsApp. You message in Teams → OpenClaw routes to Ollama → sends the reply back. You never touch Ollama directly.

This does NOT use Claude, ChatGPT, or any paid AI. The models are completely separate, free, open-source. No data is sent to Anthropic, OpenAI, or any external company.

FAQ

Common Questions

"Why is it called RadMax?" — That's the bot's display name in Teams. Under the hood it's OpenClaw + Ollama. RadMax is the friendly face; the plumbing behind it is the stack on the previous slide.

"Is it a server?" — Yes, functionally. It runs continuously, listens for requests, and serves responses. But it's just a small box on the network running two pieces of software, likely managed via DockerSoftware that packages apps into "containers" — isolated boxes that include everything the app needs. Easy to start, stop, update..

"Is it remote desktop?" — No. Nobody logs into the Mac Mini. You interact through Teams only. The admin may occasionally SSHSecure Shell — remote terminal access to a computer over the network, used for admin/maintenance. in to update models, but end users never touch it.

"How would it know our current WIP / project status?" — It doesn't. It has no connection to monday.com, email, SharePoint, or any internal system. You paste in whatever context it needs — a supplier email, a spec, your current status list. Think of it as a fast colleague who knows general knowledge but needs a briefing on current state every time.

Key Concept

Does It Learn? No.

An LLMLarge Language Model — the AI model itself. "Llama 3 70B" is an LLM. Ollama runs LLMs locally.'s knowledge is baked in during training — a massive process that happens once. The model is then frozen. Your conversations are not fed back in. When a chat ends, it's gone — no memory, no learning.

This is identical to Claude and ChatGPT at their core — they don't learn from individual conversations either.

So what's the point of keeping it local? Not "training on company data" — it's about: (1) cost — zero per message, (2) privacy — prompts never leave the building, (3) availability — works even if internet drops.
Can it be given company knowledge? Yes — via RAGRetrieval-Augmented Generation — attaching relevant documents to the prompt so the AI can reference them without retraining., not learning. Documents are attached to the prompt at query time. OpenClaw's "skills" feature can automate this. Requires technical setup to configure document indexing.

Use Cases

When to Use RadMax (Local)

Remember: RadMax has no internet and no connection to internal systems. You paste in whatever context it needs each time.

Quick Supplier Comms

How: Paste the supplier's email into RadMax and say "Draft a reply confirming revised lead time of 6 weeks, keep it professional." Copy the draft into Outlook and tweak. No setup needed — works out of the box.

LOCAL — free, fast

Meeting Agenda Structuring

How: Tell RadMax the topics — "Structure an agenda: WIP delays on X, QC flag on Y, display timeline." You supply the specifics — it doesn't know your current WIP. It's a formatting assistant, not an information source.

LOCAL — free, fast

General Knowledge Lookups

How: Ask directly — "What's the difference between PVD and electroplating?" Uses the model's training data (general/public info). No company data needed. Works instantly.

LOCAL — free, fast

Rewriting & Translation

How: Paste text and instruct — "Make this more professional" / "Shorten to 3 bullets" / "Translate from Mandarin." Pure text transformation. No system access needed.

LOCAL — free, fast

Brainstorming

How: Prompt freely — "10 ways to reduce packaging waste on towel rail shipments." The AI draws on general knowledge. You curate the results. It's a thinking partner, not a data source.

LOCAL + RAG (advanced)

Company Knowledge Bot (future)

How: Requires extra setup. OpenClaw "skills" or a RAG pipeline indexes specific docs (SOPs, specs, price lists) and auto-attaches them to prompts. Needs someone technical to configure.

Use Cases

When to Use Claude (Cloud)

CLOUD — use Claude

Complex Document Drafting

How: Use Claude directly (web, desktop, or Cowork). Upload reference docs, paste specs, ask it to draft SOPs, QC procedures, compliance docs. Claude's reasoning is significantly stronger — local models hallucinate more on detailed technical content.

CLOUD — use Claude

Code, Automation & Integrations

How: Use Cowork or Claude Code. Claude connects to external services via MCP (monday.com, Slack, etc.) and is significantly better at multi-step reasoning required for building automations and scripts.

CLOUD — use Claude

Long Document Analysis

How: Upload supplier drawings, compliance reports, or standards to Claude. Its context windowThe maximum text the model can "see" at once — prompt + history + response. Bigger = longer documents. (200K+ tokens) handles entire documents that would overflow local models (8K–32K). Ask it to cross-reference, summarise, flag issues.

EITHER — judgement call

Formatting & Restructuring

How: Paste content + describe the format — "Make this a table with item, status, owner columns." Simple formatting = RadMax (free). Complex restructuring needing comprehension = Claude (smarter).
